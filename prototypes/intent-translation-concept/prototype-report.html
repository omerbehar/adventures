Prototype Report — Intent Translation

The Magistrate scene · concept kill-criterion for Adventures

Date: 22 June 2026  ·  Path: HTML prototype + headless Node harness  ·  Classifier: Claude Haiku (claude-haiku-4-5)  ·  Status: Playtested

Verdict: PIVOT. The core idea is sound, but the first mechanisation of “classify-don’t-score” is not stable enough and the decisive-move mapping is wrong. Do not spend on content yet — apply the two-line fix below and re-test.

1. What this prototype is

A throwaway build that answers one question from the game concept’s production plan: does the “classify-don’t-score” translation pipeline keep a social scene’s decisive-move threshold stable run-to-run? This is the concept’s explicit prototype kill-criterion — the cheapest prove-or-kill of the central technical bet, to be settled before any design, tooling, or content spend.

The scene is the Magistrate: the player stands accused before a corrupt judge and must, in their own freeform words, get acquitted. The intended decisive move is to discover and then invoke the leverage that the magistrate took bribes — which should collapse a hard persuasion threshold to a trivial one (Pillar 3, “the decisive move”).

Falsifiable hypothesis

If a player expresses the decisive move in their own varied words, the pipeline collapses the threshold consistently. Accept if, across N≈20 runs of ~5 natural phrasings, the decisive-move outcome holds ≥95%, and the deterministic stages are 100% reproducible given a fixed classification.

Riskiest assumption (tested first): that an LLM classifying evaluative/social intent is stable enough run-to-run that the threshold does not flip.

3. How it was run

  • Deterministic half — run headlessly with no key: a 1000-run determinism self-test, class-divergence check, and the discover-then-leverage mechanic.
  • The real bet — run against live Claude Haiku (claude-haiku-4-5), strict tool use: a probe printing the raw classification 5× per phrasing, plus the stability harness (N=20, Diplomat, scandal already discovered).

4. Results

4.1 Deterministic half — PASS

  • Stage B determinism: PASS for all three classes (1000 runs each → byte-identical vector).
  • Class divergence works: one classification → Diplomat win, Scholar win (collapsed threshold), Assassin advance (not a persuader).
  • Decisive-move mechanic works: discover → reveal the scandal facet → leverage → threshold collapses → win.

This confirms the design intent of classify-don’t-score: it successfully quarantines all run-to-run variance into Stage A, which is exactly where the harness then caught it.

4.2 The real bet (Stage A stability) — the kill-criterion

Stage A axis classification is NOT stable run-to-run. Probing each phrasing five times, four of five produced two or three different (axis / channel / ordinal) classifications. The same sentence was labelled with different scalar axes on different runs.

Player phrasing (the decisive move, varied)Distinct classifications in 5 runsScandal facet detected
“I remind him, quietly, that I know about the bribes”3 — social/intimidation, insight/deception, insight/intimidationtrue ×5
“I let slip that the scandal could become public”3 — social, force, and stealth (all intimidation)true ×5
“If he convicts me, everyone hears how he was paid off”1 — social/intimidation/majortrue ×5
“I hint his corruption need not come to light”2 — deception, intimidationtrue ×5
“I calmly mention the gold that changed hands”3 — insight/none, social/persuasion, insight/persuasiontrue ×5

But the leverage facet was detected with perfect stability: invokesScandalFacet = true in all 25 classifications. The model is dependable about what leverage is invoked; it is unreliable about which scalar axis the action is.

The apparent “100% stable” outcome was misleading. The N=20 stability run reported 100% for the first phrasings — but only because every divergent classification happens to map to advance (a loss of the decisive move) for the Diplomat: scandal leverage reads as intimidation/deception, where the Diplomat is weak, so nothing clears a win threshold. Where a winning classification was even possible (“I calmly mention the gold…”: persuasion 1/5 → win, otherwise advance), the outcome flips — roughly 80%, i.e. unstable. So the decisive move both (a) rarely lands and (b) when it can, depends on a coin-flip axis label.

4.3 Operational finding

Under repeated calls the run aborted with API 400: Grammar compilation timed out — strict tool-use grammar compilation intermittently times out. A production translator needs retry/backoff or a non-strict + repair path (relevant to ADR-0007’s structured-output discipline).

5. Best / worst / surprise

  • Best: facet recognition was rock-solid (25/25). The model is dependable at naming the leverage — which is the part the decisive move actually needs.
  • Worst: single-axis classification of social intent is genuinely noisy — the exact variance the concept feared.
  • Surprise: the scene masked the instability. Stability-by-degenerate-collapse (everything fails the same way) looked like a pass until the probe exposed the underlying variance.

6. Verdict and the fix

The core idea survives, but the first mechanisation does not meet the kill-criterion as written. The fix is specific and cheap:

  1. Key the decisive-move collapse on the reliably-detected facet (invokesScandalFacet), not on a brittle single primary axis or magnitude. The model is stable about the facet; lean on that.
  2. Let Stage A name multiple axes with per-axis ordinals (as ADR-0004 already envisions). A single primary axis discards nuance and lets incidental label variance flip outcomes.
  3. Handle the strict-tool grammar 400 with retry/backoff or a non-strict + JSON-repair path.

Re-run the same harness after changes (1)+(2) before re-deciding. No content, tooling, or Compiler spend until the decisive-move outcome is stable.

7. What this cost

~30 live Haiku classifications — a fraction of a cent. The prototype paid for itself by catching, in one afternoon, a stability-and-mapping flaw that would otherwise have surfaced after content authoring began.

Throwaway prototype — not production code. The production translator is written fresh from the accepted ADRs; prototype code is never refactored into the engine.
